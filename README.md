# Пересечение отрезков в трехмерном пространстве
Программа для установления пересечения и поиска точки пересечения отрезков в трехмерном пространстве.
Представлен графический интерфейс, где есть возможность наглядно увидеть, как располагаются отрезки в пространстве. В графическом интерфейсе есть возможность выбрать представление в двухмерном пространстве.
Также представлена функция для поиска точки пересечения в двухмерном пространстве(сначала решил реализовать двумерный случай и в последствии решил оставить)

Программа тщательно протестирована, реализованы тесты со 100 покрытием кода.

## Алгоритм пересечения 3D-отрезков

В программе реализован алгоритм нахождения пересечения двух отрезков в трёхмерном пространстве. Каждый отрезок задаётся двумя точками:
отрезок AB определяется точками A(x1, y1, z1) и B(x2, y2, z2),
отрезок CD определяется точками C(x3, y3, z3) и D(x4, y4, z4).

Отрезки можно записать в параметрической форме:

AB(t) = A + t(B − A),  t ∈ [0,1]
CD(s) = C + s(D − C),  s ∈ [0,1]

Пересечение существует, если существуют такие параметры t и s, что выполняется равенство:

A + t(B − A) = C + s(D − C)

Алгоритм решает эту задачу в несколько этапов.

Сначала обрабатываются вырожденные случаи. Проверяется длина каждого отрезка. Если квадрат длины отрезка близок к нулю (|AB|² < EPS или |CD|² < EPS), это означает, что соответствующий отрезок выродился в точку. В таком случае задача упрощается. Если оба отрезка являются точками, проверяется совпадают ли они. Если точки совпадают, это и есть точка пересечения, иначе пересечения нет. Если только один из отрезков является точкой, проверяется лежит ли эта точка на втором отрезке. Для этого точка проецируется на прямую второго отрезка и вычисляется параметр u = ((A − C) · (D − C)) / |CD|². Если 0 ≤ u ≤ 1, точка лежит на отрезке и является точкой пересечения.

Если оба отрезка имеют ненулевую длину, вычисляются их направляющие векторы:

u = B − A
v = D − C

Затем вычисляется их векторное произведение:

n = u × v

Если длина этого вектора близка к нулю, то направляющие векторы параллельны, а значит прямые либо параллельны, либо лежат на одной прямой. Для различения этих случаев вычисляется векторное произведение (A − C) × u. Если оно также близко к нулю, то точки A, B, C, D лежат на одной прямой. В этом случае задача сводится к поиску перекрытия двух отрезков на этой прямой. Возможны три результата: отрезки не пересекаются, пересекаются в одной точке или пересекаются на некотором участке.

Если прямые не параллельны, проверяется лежат ли они в одной плоскости. Для этого используется скалярное тройное произведение:

(A − C) · (u × v)

Если его абсолютное значение больше EPS, прямые являются скрещивающимися и не могут пересекаться.

Если прямые не параллельны и компланарны, можно вычислить параметры пересечения. Для этого используются формулы:

t = ((C − A) × v) · (u × v) / |u × v|²
s = ((C − A) × u) · (u × v) / |u × v|²

Полученные параметры проверяются на принадлежность интервалу [0,1]. Если t и s находятся в этом диапазоне, точка пересечения лежит на обоих отрезках. В противном случае прямые пересекаются вне границ отрезков и пересечения отрезков нет.

Если пересечение существует, координаты точки находятся из параметрического уравнения:

P = A + t(B − A)

Во всех вычислениях используется небольшое значение EPS, позволяющее учитывать погрешности вычислений с плавающей точкой.

Функция возвращает результат в виде std::pair<Vector3D, Vector3D>. Если пересечение происходит в одной точке, возвращается пара {P, P}. Если отрезки перекрываются, возвращаются крайние точки пересечения {P1, P2}. Если пересечения нет, выбрасывается исключение.

## Тесты
Для проверки корректности реализации используются модульные тесты. Тесты покрывают все основные геометрические случаи пересечения отрезков в трёхмерном пространстве.
Проверяются следующие основные случаи:

1. Обычное пересечение — отрезки пересекаются в одной точке внутри обоих отрезков

2. Пересечение в конечной точке — пересечение происходит в одной из граничных точек отрезка

3. Отсутствие пересечения — отрезки лежат в одной плоскости, но не пересекаются

4. Параллельные отрезки — направляющие векторы параллельны, пересечения нет

5. Скрещивающиеся отрезки — отрезки не лежат в одной плоскости

6. Коллинеарные отрезки — отрезки лежат на одной прямой

7. Перекрывающиеся отрезки — отрезки частично совпадают

8. Вырожденные случаи — один или оба отрезка являются точками

При сравнении координат используется небольшое значение EPS, позволяющее учитывать погрешности вычислений с плавающей точкой.

Тесты обеспечивают 100% покрытие кода
